Template Library

The Template Library is a catalogue of approved page layouts created in WordPress and shared here for reuse across Performance projects.

Templates are stored in GitHub as cleaned, reusable reference implementations. Use these templates to copy structure, layout, and supporting code when building new pages.

How Templates Are Used

  1. Templates are created and validated in WordPress projects.
  2. The reusable structure (HTML, CSS, JS) is added to GitHub.
  3. Templates are listed here so the team can discover and reuse them.

GitHub is the source of truth for copying templates. WordPress pages may be linked as live examples, but code should always be copied from GitHub.

How to Use a Template

  1. Select a template that matches your page type.
  2. Open the template’s GitHub link.
  3. Copy the sections or full layout you need.
  4. Adapt the content, styling, and scripts for your project.

Landing Pages

Campaign and promotional layouts with clear messaging, calls to action, and supporting sections.

Business Pages

Standard multi-section layouts for services, departments, and informational content.

Forms & Lead Capture

Page layouts focused on form submission, lead generation, and user input.

Content Pages

Blog-style and editorial layouts designed for longer-form content and readability.

Dashboards & Utilities

Internal or utility-style layouts that support data display, tools, or workflows.

Adding a Template

Team members are encouraged to contribute templates that represent reusable, real-world page layouts.

  • Templates should originate from an actual WordPress page.
  • Remove site-specific content and unnecessary dependencies.
  • Add the cleaned template to the Templates folder in GitHub.
  • Optionally include a link to the original WordPress page as a live reference.

Follow the contribution guidelines to keep templates consistent, readable and useful for the entire team.
