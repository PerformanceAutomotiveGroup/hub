Module 1: Directory Pipelines & Root Isolation

1.1 Absolute Path Mapping Specification

The Performance Hub runs as an interconnected, multi-tier static ecosystem. Relative file traversals (such as ../../nav/head.html) are strictly prohibited. Moving a file between directory branches will alter its context and break references. All linkages must execute via root-relative mapping pipelines absolute to the domain root.

Pipeline Syntax Constraints:
❌ <script src="../js/include.js"></script>
✅ <script src="/hub/js/include.js"></script>

1.2 Sandbox Code Decoupling Framework

To safely capture structural layouts without causing parent window asset conflicts, code blocks are strictly isolated. Components must be contained in native HTML <template> nodes with precise identifier hooks. The workspace rendering engine extracts text matrices out of these templates, parses them through security filters, and injects clean text outputs directly into the Prism syntax highlighter elements.

<template id="html-template">
<!-- Raw structural markup components here -->
</template>

<template id="css-template">
/* Clean layout formatting presentation code here */

// Functional behaviour logical configurations here
</template>

Module 3: Scaffolding Chapters & Deep Subpages

3.1 Spawning a New Sub-Category Chapter Hub

When expanding our documentation roadmap with a brand-new code discipline chapter (e.g., creating a new chapter track inside the CSS Library), follow this exact execution sequence:

  1. Navigate into your local category directory target path (e.g., /hub/pages/CSS/).
  2. Spawn a new, lowercase hyphen-separated chapter directory subfolder (e.g., /hub/pages/CSS/grid-layouts/).
  3. Inside that new path, establish a clean index baseline control page named exactly index.html.
  4. Configure the new page's metadata tracking hooks to point right back to its shared sidebar navigation partial include file: <div data-include="/hub/nav/csssidenav.html"></div>.
